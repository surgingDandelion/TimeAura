# TimeAura UI 模型最终设计提示词

## 1. 使用说明

本文件用于直接提供给 UI 设计模型，帮助模型基于 TimeAura 的 PRD、低保真线框和 HTML 原型，稳定输出高保真桌面应用视觉方案。

建议使用方式：

- 将本文件作为主提示词
- 同时附带以下参考文件给模型：
  - `docs/TimeAura-PRD-v1.md`
  - `docs/TimeAura-LowFi-Wireframes-v1.md`
  - `index.html`
- 若模型支持多文件输入，优先保证它能看到 PRD 和低保真线框
- 若模型只能接收一段文本，可直接复制本文件中的“最终提示词”部分

---

## 2. 输入背景摘要

- 产品名称：TimeAura
- Slogan：让每个重要时刻如约而至
- 产品类型：AI 备忘录桌面应用
- 平台方向：桌面端优先，风格参考 mac 原生专业工具
- 核心能力：
  - 备忘录与待办统一为同一个 Record 列表
  - 标签与分类统一为同一套体系
  - 单行快速新增
  - 模糊检索
  - 优先级、状态、排序、时间管理
  - 提醒条与批量改期
  - Markdown 编辑 / 分栏 / 预览
  - AI 报告生成（周报 / 月报 / 自定义）
  - AI 通道配置（含 Anthropic 协议）
  - 浅色 / 暗色主题切换

请基于现有产品定义，输出一套可直接进入产品设计或工程实现的高保真桌面 UI 方案，而不是概念海报式设计稿。

## 4. 最终提示词

你是一名资深桌面端产品设计师与 mac 专业工具 UX/UI 设计师，请为 TimeAura 设计一套高保真桌面应用 UI。

项目背景如下：

- 产品名称：TimeAura
- Slogan：让每个重要时刻如约而至
- 产品定位：一个专注于“待办 + 备忘 + AI 总结”的桌面效率应用
- 使用场景：快速记录、时间提醒、批量改期、Markdown 内容沉淀、周报月报生成、AI 通道配置

请严格遵守以下信息架构与体验约束：

1. 整体结构

- 采用三栏桌面布局：
  - 左侧简洁导航栏
  - 中间主列表区
  - 右侧详情 inspector
- 右侧详情默认收起，点击列表项后再展开
- 中间与右侧之间保留可拖拽分割条

2. 左侧导航栏

- 保留系统视图：今天、计划、全部、已完成
- 标签与分类合并成一套标签体系，直接展示在左侧，不额外做“标签功能区”大标题
- 每个系统项与标签项显示待办数量
- 左下角仅保留纯图标按钮：AI 报告、通道配置、快速新增
- 左侧保持克制、纯净，不在这里放重操作按钮

3. 中间主列表

- 顶部必须紧凑，优先给列表留出垂直空间
- 不要在主列表顶部常驻大块快速新增输入框
- `快速新增` 通过按钮触发悬浮弹层，弹层内提供单行输入并支持连续录入
- 保留搜索、筛选、排序
- 列表上方需要有提醒条，用于提示即将过期、逾期、积压任务
- 提醒条支持展开命中任务，并支持仅对选中部分一键改期
- 列表采用单行高密度结构，不要改成卡片瀑布流
- 每条记录至少表现：
  - 完成勾选
  - 标题
  - 标签
  - 时间
  - 优先级
  - 快捷操作
- 列表的 hover、active、selected、done 状态需要清晰区分

4. 右侧详情 inspector

- 整体必须像专业桌面工具的 inspector 属性面板，不像后台系统表单
- 字段按分组组织，分隔线轻、层级明确
- 属性区包含：
  - 状态
  - 优先级
  - 标签
  - 时间
- 内容区包含 Markdown 编辑器，支持：
  - 编辑
  - 分栏
  - 预览
- AI 摘要、润色、管理标签等按钮统一为 inspector 次级操作语言
- 标签管理入口在详情区，不在左侧导航 hover 出现

5. AI 报告页

- 左侧为条件区，右侧为结果编辑 / 预览区
- 条件区需要包含：
  - 报告类型
  - 时间范围
  - 标签范围
  - 状态范围
  - 模板选择
  - 模板配置入口
  - AI 通道选择
- 右侧结果区要更像 Markdown 文稿工作台，而不是普通表单区
- 顶部操作包含：重新生成、复制、保存为记录、导出 Markdown

6. AI 通道配置页

- 左侧通道列表，右侧配置表单
- 列表项需展示：
  - 通道名称
  - 服务商类型
  - 启用状态
- 表单需支持：
  - 通道名称
  - 服务商类型
  - Base URL
  - API Key
  - 模型名称
  - 温度
  - 最大输出
  - 超时
  - 系统 Prompt
  - 默认语言
  - 启用状态
- 页面中明确支持 Anthropic 协议
- 测试连接按钮必须清晰可见

7. 视觉风格要求

- 风格方向：mac 原生专业工具感、克制、冷静、轻专业
- 不是后台系统，不是 marketing 页面，不是赛博科技风
- 不要紫色主视觉
- 不要大面积高亮科技蓝
- 不要过度玻璃拟态、过重发光或炫光
- 浅色主题要干净、安静、有专业感
- 暗色主题要冷静、细腻、保持桌面原生感

8. 视觉 token 方向

- 主蓝：低饱和时间蓝，用于焦点、选中、主按钮
- 提醒色：暖橙
- 风险色：柔和红
- 完成色：自然绿
- 文本以深灰蓝 / 浅灰蓝为主
- 阴影要克制，仅用于建立层级
- 分隔线细且统一
- 圆角连续、统一

可参考的关键语义色：

- accent：`#5f8fdc`
- warning：`#d88e52`
- danger：`#d56d73`
- success：`#4b9a71`
- neutral：`#97a6b8`

9. 排版与控件要求

- 字号层级稳定，不要忽大忽小
- 标题以 700 为主，普通按钮与标签多用 600
- 不建议大面积使用 800 粗体
- select、date、checkbox、segmented control 都要像真实产品实装控件
- hover / focus / active / selected / disabled 都必须有真实状态
- 禁用态不能只是简单降低透明度，要保留结构感

10. 关键交互要求

- 点击 `快速新增` 后必须打开悬浮弹层，并让输入框立即获得焦点反馈
- 列表行 hover 反馈轻，不夸张
- 当前打开详情的行是 active 态
- 批量选择的行是 selected 态
- 提醒条展开后能看到命中任务和已选数量
- 详情区可主动收起
- 明暗主题切换时布局结构保持一致

11. 输出要求

- 输出完整的高保真页面方案说明
- 至少覆盖：
  - 主工作台
  - 提醒条展开态
  - 右侧详情区
  - 标签管理
  - AI 报告页
  - AI 通道配置页
  - 浅色主题
  - 暗色主题
- 如需展示组件规范，请额外给出：
  - 按钮规范
  - 字段行规范
  - 列表行规范
  - segmented control 规范
  - checkbox / 完成勾选规范

请输出结果时尽量体现“真实桌面产品可落地感”，而不是只做一张漂亮的概念稿。

## 5. 如果模型支持更细颗粒度输入，可追加的补充约束

### 5.1 给视觉模型的补充

- 滚动条统一、低存在感、hover 增强
- 分割条要有明确可拖拽暗示
- 右侧详情区的字段行更像 inspector，而不是默认 input form
- 暗色主题下注意对比度，不允许浅色字叠浅色底

### 5.2 给交互模型的补充

- 主列表必须体现 hover / active / selected / done / disabled 五类状态
- 快速新增、提醒条展开、批量改期弹层、主题切换都要给出状态变化
- 标签管理要区分“当前记录标签”和“标签库”两层结构

### 5.3 给工程导向模型的补充

- 输出时优先考虑桌面应用可实现性
- 避免依赖过于复杂、难还原的特殊材质
- 组件状态与 token 要尽量可复用
