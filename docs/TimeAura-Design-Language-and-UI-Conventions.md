# TimeAura 设计语言与 UI 规约

## 1. 文档目的

本文件是 TimeAura 当前阶段的统一视觉与交互实现规约，服务于以下对象：

- UI 设计师
- 前端开发
- AI 设计模型
- AI 编码模型

目标不是描述“审美方向”，而是提供一套可直接执行的界面标准，减少后续反复精修。

---

## 2. 设计语言总原则

### 2.1 产品气质

- 接近 mac 原生专业工具，而不是网页后台
- 冷静、克制、轻专业，不做夸张营销感
- 信息密度偏高，但不拥挤
- 通过层级、留白、边线和轻状态表达专业感，而不是靠重阴影和大圆角

### 2.2 禁止偏离项

- 不使用紫色作为主视觉
- 不使用强烈霓虹蓝、荧光色、赛博配色
- 不把主工作台做成卡片瀑布流
- 不在列表、详情、配置页混用圆角语言
- 不把 inspector 区做成普通后台表单
- 不让 hover / active / selected 三种状态难以区分
- 不为了“设计感”牺牲可读性和密度

---

## 3. 视觉基线

### 3.1 字体体系

- 默认字体栈：
  - `"SF Pro Text", "PingFang SC", "Helvetica Neue", sans-serif`
- 中文优先使用系统原生观感，不额外引入花哨字体
- 标题字重不超过 `650`
- 常规正文与字段值以 `400 ~ 600` 为主，不使用大面积粗黑字

### 3.2 字号层级

- 窗口级标题：`22px ~ 24px`，`650`
- 页面标题：`18px ~ 20px`，`650`
- 区块标题：`13px ~ 14px`，`600 ~ 700`
- 主列表标题：`13px ~ 14px`，`600`
- 表单标签 / inspector 字段名：`11px ~ 12px`，`600`
- 辅助文案 / meta：`11px ~ 12px`，`400 ~ 500`
- 极小辅助信息不得低于 `10.5px`

### 3.3 行高标准

- 页面标题：`1.15 ~ 1.2`
- 列表标题：`1.3 ~ 1.35`
- 表单 / 字段：`1.25 ~ 1.4`
- 辅助文案：`1.35 ~ 1.55`

### 3.4 字重规则

- 页面标题：`650`
- 区块标题：`600 ~ 700`
- 列表标题：`600`
- 主按钮文案：`600`
- 表单内容：`500`
- 次级说明：`400 ~ 500`

## 4. 布局规则

### 4.1 整体壳层

- 桌面端主工作台使用左侧导航 + 中间主列表 + 右侧 inspector 三栏结构
- 窗口内容必须贴满应用窗口，不再额外模拟系统外边框留白
- 除弹层外，主界面不允许页面级滚动；滚动必须落在各自独立区域内部
- 左侧导航高度始终与窗口等高，不被中间内容撑开

### 4.2 三栏宽度建议

- 左侧导航：
  - 常规宽度 `232px ~ 248px`
- 中间列表：
  - 优先占据剩余空间
- 右侧详情：
  - 默认收起
  - 展开后建议 `336px ~ 420px`
  - 支持拖拽分栏

### 4.3 间距体系

- 壳层外层间距：`16px ~ 18px`
- 页面区块间距：`10px ~ 14px`
- 面板内部常规间距：`10px ~ 12px`
- 字段组间距：`8px ~ 10px`
- 紧凑控件水平间距：`6px ~ 10px`
- 微型操作与图标间距：`4px ~ 8px`

### 4.4 小屏幕适配

- 左侧导航内部模块优先压缩为高密度布局，不允许文字换行变形
- 中间工具栏在小窗口下优先压缩操作组，不允许重要搜索与筛选被挤没
- reminder / search / filter 区必须自适应宽度，不允许固定宽度导致断裂和换行

---

## 5. 形状、边线与阴影

### 5.1 圆角规则

- 当前正式产品语言以“直角主体 + 小圆角弹层”为准
- 左侧导航、中间列表、右侧详情、回收站、通道配置主体区域：
  - 不使用大圆角
  - 内部区块保持统一直角或极小圆角
- 悬浮弹层 / modal / sheet：
  - 使用小圆角，建议 `12px ~ 16px`
- 行内芯片 / 状态胶囊：
  - 允许胶囊圆角

### 5.2 边线规则

- 优先使用低对比度边线建立结构
- 浅色模式边线：
  - `rgba(15, 23, 42, 0.06 ~ 0.10)`
- 暗色模式边线：
  - `rgba(173, 192, 219, 0.07 ~ 0.12)`
- 分隔线只用于分组，不用于制造视觉噪音

### 5.3 阴影规则

- 常规页面面板不依赖明显阴影撑层级
- hover 阴影应极轻，仅作为空气感补充
- 暗色模式尽量减少发蓝发亮的外阴影
- 只有弹层、浮层、上下文菜单允许使用明确阴影

## 6. 颜色系统

### 6.1 Light Theme

- 主背景：`#eef1f5`
- 页面渐变基底：
  - `#f7f8fa -> #eceff4`
- 主内容背景：
  - `rgba(255, 255, 255, 0.86 ~ 0.95)`
- 主文字：`#1f2933`
- 次级文字：`#66788a`
- 弱化文字：`#7b8d9f`
- 边线：`rgba(15, 23, 42, 0.08)`

### 6.2 Dark Theme

- 主背景：深冷蓝灰，不偏紫
- 常规面板背景：
  - `rgba(19, 28, 41, 0.86 ~ 0.94)`
- 弹层背景：
  - `rgba(17, 24, 35, 0.98)` 到 `rgba(23, 31, 45, 0.985)` 的轻微纵向渐变
- 主文字：`#e8eef8 ~ #eef4fd`
- 次级文字：`#9fb0c8`
- 弱化文字：`#8fa4bf ~ #90a4be`
- 边线：`rgba(173, 192, 219, 0.07 ~ 0.10)`

### 6.3 语义色

- 选中主蓝：
  - 浅色：`rgba(95, 143, 220, 0.06 ~ 0.10)`
  - 暗色：`rgba(124, 164, 236, 0.08 ~ 0.14)`
- 风险红：
  - 用于删除、P1、错误
- 提醒橙：
  - 用于即将到期、提醒
- 完成绿：
  - 用于已完成、成功状态
- 中性灰：
  - 用于 P4、次级状态、非关键描述

### 6.4 优先级色

- `P1`
  - 浅色：柔和红底 + 深红字
  - 暗色：低饱和红底 + 浅红字
- `P2`
  - 浅色：暖橙底 + 棕橙字
  - 暗色：柔暖橙底 + 浅橙字
- `P3`
  - 浅色：低饱和蓝底 + 深蓝字
  - 暗色：低亮蓝底 + 浅蓝字
- `P4`
  - 浅色：灰底 + 深灰字
  - 暗色：灰蓝底 + 浅灰字

要求：

- 同一优先级的颜色语义必须同时用于列表、筛选器、详情和提醒链路
- 暗色模式不得出现“默认蓝底输入框”或莫名高亮底

## 7. 控件规范

### 7.1 按钮

- 主按钮：
  - 仅用于当前页面最关键动作
  - 不要一屏出现多个同级主按钮
- 次级按钮：
  - 使用轻边线或轻底色
  - 更接近 mac inspector 的次级操作语言
- 纯图标按钮：
  - 建议 `28px ~ 32px`
  - hover 才明显，默认尽量轻

### 7.2 输入框 / Select / Date

- 高度建议：
  - 常规 `30px ~ 36px`
  - inspector 紧凑场景 `28px ~ 32px`
- 文本字号：
  - `12px ~ 13px`
- 默认态避免过重内阴影
- focus 用边线和轻外发光表示，不使用夸张蓝边
- 暗色模式输入框背景需接近面板本体，不单独发蓝

### 7.3 Segmented Control

- 用于 Markdown 编辑模式切换、协议切换等强相关多态操作
- 视觉必须更接近原生分段控件，而不是网页 tab
- 选中态以底板和文字对比表达，不依赖重阴影

### 7.4 Checkbox

- 非必要场景不出现
- 在工作台列表中，优先用行选中底色和快捷键表达选中，不额外摆一个难看的复选框

### 7.5 色点选择

- 列表/标签颜色选择优先使用小圆点，不使用大色块
- 颜色输入在编辑行中右对齐
- 色点尺寸建议：
  - `16px ~ 20px`

---

## 8. 组件级规则

### 8.1 左侧导航

- 视图与“我的列表”使用同一套密度语言
- 角标数字对比度弱于标题，不喧宾夺主
- 选中态使用轻底色，不使用厚重描边
- hover 只轻微提亮
- 左下角功能图标描边、尺寸和激活底板必须统一

### 8.2 中间主列表

- 列表项使用单行高密度结构
- 左侧优先级必须比右侧更容易被看见
- 右侧只保留状态和时间等收口信息
- 行 hover、selected、focused 三者要可区分，但强度依次递进
- 中间列表必须有独立滚动条

### 8.3 右侧 Inspector

- 要像专业桌面工具的属性面板，不像普通表单区
- section 之间通过轻分隔线、间距、字段密度分组
- 字段行高度要紧凑，让更多核心字段进入首屏
- 操作按钮统一为次级工具语言

### 8.4 我的列表 / 标签管理

- 外圈弹层可小圆角，内部遵循当前主应用直角语言
- 编辑区与列表库双栏布局，左窄右宽
- 颜色点右对齐
- 编辑说明只保留一行轻提示
- 删除操作降级为次级危险操作，不要一堆强调按钮

### 8.5 AI 通道配置 / AI 报告

- 必须与工作台保持同一套壳层语言
- 列表列、详情 inspector、测试面板纵向节奏一致
- 暗色模式避免局部过蓝、过亮、发光输入框
- segmented / select / checkbox 要比普通网页表单更原生

---

## 9. 交互状态强度规范

### 9.1 Hover

- 只做轻微亮度或底色变化
- 不允许大面积高饱和蓝色 hover

### 9.2 Active / Pressed

- 比 hover 稍强，但不应比 selected 更显眼

### 9.3 Selected

- 以底色、边线、文字对比共同表达
- 列表选中不依赖 checkbox

### 9.4 Focus

- 优先用清晰但克制的 focus ring
- 暗色模式下 focus ring 仍需低饱和

### 9.5 Disabled

- 通过透明度、文字弱化、背景去激活表达
- 不允许 disabled 仍保留明显 hover / shadow

---

## 10. 滚动条与分割线

- 滚动条必须轻、细、系统感强
- 默认透明轨道
- thumb 为低对比度圆角胶囊
- 暗色模式下滚动条不发亮、不泛蓝
- 分割线强度低于边线，只用于结构提示

---

## 11. AI 生成与实现约束

后续无论让设计模型还是编码模型生成页面，都必须遵守以下约束：

- 先继承本文件，再继承 `index.html` 当前高保真原型
- 优先复用已存在的字体、颜色、间距、控件密度
- 不得擅自引入另一套按钮语言
- 不得擅自把直角主体改回大圆角卡片
- 不得把深色模式做成偏紫、偏荧光的科技风
- 不得为了“高级感”增加无必要阴影和描边
- 新页面若没有明确设计稿，默认按“工作台壳层 + inspector 详情”语言落地

---

## 12. 规范落地顺序

后续做新页面或重构旧页面时，按以下顺序判断：

1. 先看本规约，确认视觉语言与密度边界
2. 再看 `docs/TimeAura-PRD-v1.md`，确认页面功能与交互规则
3. 再看 `docs/TimeAura-LowFi-Wireframes-v1.md`，确认结构骨架
4. 最后参考 `index.html` 与当前桌面端已实现页面，做像素级贴合
