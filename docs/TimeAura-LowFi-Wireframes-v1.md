# TimeAura 低保真线框图说明（V1）

## 1. 文档目标

本稿用于定义 TimeAura V1 的核心页面骨架与关键交互，帮助后续进入 UI 设计、原型制作和前端开发阶段。

本次重点覆盖：

- 左侧栏
- 主列表
- 提醒条
- 右侧详情区
- AI 报告页
- AI 通道配置页

## 2. 设计原则

- 简约优雅：页面元素少而准，不做复杂导航堆叠
- 时间优先：即将过期、逾期、积压事项始终有更高视觉优先级
- 快速录入：新增入口始终可见，支持连续录入
- 单行浏览：主列表高密度但不拥挤，适合快速扫视
- 渐进展开：先看标题与状态，再在右侧补细节

## 3. 页面总览

TimeAura 采用三栏式桌面布局：

1. 左侧栏：系统视图 + 标签列表 + 底部功能按钮
2. 中间主列表：快速新增、搜索筛选、提醒条、单行记录列表
3. 右侧详情：记录内容编辑、Markdown 预览、属性设置

### 4.1 布局说明

- 左侧栏宽度建议 220-260px，保持克制
- 中间主列表为主工作区，宽度最大
- 右侧详情固定存在，适合桌面高频编辑场景
- 底部功能按钮独立于侧栏内容区，避免视觉拥挤

### 4.2 左侧栏说明

#### 结构

- 顶部：品牌名与 slogan
- 中部上方：系统视图
- 中部下方：标签列表
- 底部：AI 报告、回收站、设置

#### 系统视图建议

- 今天
- 计划
- 全部
- 已完成

#### 标签展示规则

- 不显示“标签管理”标题
- 每条标签右侧显示未完成数量
- 标签支持颜色圆点
- 点击标签直接切换主列表上下文

#### 交互说明

- 当前选中项使用浅底高亮
- 鼠标悬停标签可出现编辑入口
- 长标签自动截断，悬停显示完整名称

### 5.1 主列表关键区块

- 顶部标题区：当前视图名称 + 数量
- 快速新增区：常驻输入框
- 搜索筛选区：尽量单行显示，避免多层工具条
- 提醒条：仅在有即将过期、逾期或积压场景时出现
- 提醒条支持展开查看命中任务，并允许只对部分任务执行批量改期
- 列表区：单行信息展示
- 底部批量操作条：仅在多选后出现

### 5.2 单行列表项说明

每条记录建议保持单行高密度结构：

- 完成勾选框
- 标题
- 优先级色标
- 标签
- 截止时间
- 快捷改期按钮

### 5.3 视觉优先级建议

- 标题信息最重
- 时间状态次重
- 优先级色标需要明显但不刺眼
- 标签作为辅助信息，不抢标题视觉

### 5.4 状态样式建议

- 即将过期：时间文本高亮橙色
- 已逾期：时间文本高亮红色
- 已完成：标题变浅并加删除线
- 无时间：弱化显示

## 6. 线框 C：提醒条与一键改期

```text
+--------------------------------------------------------------------------------------+
| 提醒：今晚 18:00 前有 4 条任务重叠，其中 2 条为 P1                                  |
| [展开命中任务] [一键调整] [顺延 1 小时] [改到明天上午] [稍后提醒]                  |
+--------------------------------------------------------------------------------------+
```

### 6.1 提醒条出现条件

- 24 小时内即将到期
- 已逾期仍未完成
- 同一时间段积压多条任务
- 高优先级任务同时集中

### 6.2 提醒条行为

- 点击 `展开命中任务` 后，展开本次提醒命中的任务清单
- 用户可在展开区手动勾选部分任务
- 点击 `一键调整` 后，默认作用于“当前选中任务”；如未选中，则默认作用于全部命中任务
- 点击快捷时间按钮后立即生效
- 点击 `稍后提醒` 可临时静默，例如 30 分钟后再次提醒

### 6.3 展开命中任务线框

```text
+--------------------------------------------------------------------------------------+
| 提醒：今晚 18:00 前有 4 条任务重叠，其中 2 条为 P1                                  |
| [收起] [全选] [清空选择]                                    已选 2 / 命中 4          |
|--------------------------------------------------------------------------------------|
| ☑ 采集历史数据治理                       P1     明天 18:00                           |
| ☑ 对齐算子                               P1     已逾期                              |
| ☐ 周报草稿整理                           P2     今天 17:00                          |
| ☐ 技术方案补充                           P3     今天 18:00                          |
|--------------------------------------------------------------------------------------|
| [仅改选中] [全部顺延] [改到明天上午] [改到周五] [自定义时间]                        |
+--------------------------------------------------------------------------------------+
```

### 6.4 批量改期弹层线框

```text
+------------------------------------------------------+
| 批量调整 2 条已选任务时间                            |
|------------------------------------------------------|
| 采集历史数据治理                                     |
| 对齐算子                                             |
|------------------------------------------------------|
| 选择新的时间：                                       |
| ( ) 延后 1 小时                                      |
| ( ) 今天下班前                                       |
| ( ) 明天上午 09:00                                   |
| ( ) 本周五 18:00                                     |
| ( ) 自定义                                           |
|------------------------------------------------------|
|                                    [取消] [确认调整] |
+------------------------------------------------------+
```

### 6.5 设计重点

- 这是 TimeAura 最重要的差异化模块之一
- 交互要轻，不要做成复杂的日历调度器
- 目标是“快速挪动任务”，不是“重新规划人生”
- 用户必须能够只调整部分积压任务，而不是被迫整体顺延

- 保持结构稳定，不因字段多少频繁跳动

### 7.2 详情区交互重点

- 标题、状态、优先级、标签、时间都在上半区
- Markdown 编辑器位于视觉中心
- AI 操作放在编辑器上方工具区
- 元信息固定在底部，弱化展示

### 8.1 页面结构

- 左侧为生成条件配置区
- 右侧为报告预览与编辑区
- 页面结构要明显区别于主工作台，体现“生成结果工作台”的感觉

### 8.2 关键控件说明

- 报告类型：周报、月报、自定义
- 时间范围：必须可框选
- 模板：支持切换
- AI 通道：支持选择不同模型通道
- 生成按钮：应为页面主按钮

### 8.3 结果区要求

- 输出默认为 Markdown
- 支持直接编辑
- 支持重新生成但保留当前草稿
- 支持保存为一条新的 Record

### 8.4 AI 通道配置入口说明

- 报告页中的 `AI 通道` 下拉仅负责快速切换
- 当用户点击 `管理通道` 或从左下角设置进入时，打开独立的 AI 通道配置页
- 配置页负责新增、编辑、测试和启停通道

## 10. 页面尺寸与布局建议

### 10.1 推荐桌面布局比例

- 左侧栏：18%
- 中间主列表：42%
- 右侧详情：40%

### 10.2 最小宽度建议

- 应用最小宽度建议不低于 1280px
- 当窗口过窄时，右侧详情可切换为抽屉模式

### 10.3 视觉节奏建议

- 左侧紧凑
- 中间密集
- 右侧舒展

这样可以让“列表处理效率”和“内容编辑体验”同时成立。

## 11. 我对这版线框的建议结论

这版低保真线框的核心不是把功能全部摆出来，而是先把 TimeAura 最重要的体验骨架定住：

- 左侧极简侧栏，系统入口少但高频
- 中间列表承担主要处理效率
- 提醒条突出“即将过期任务一键调整”
- 右侧详情承接 Markdown 长内容
- AI 报告页形成独立的产出空间

如果下一步继续推进，建议直接基于这份线框出：

- 页面组件清单
- 高保真 UI 说明
- 前端页面拆分方案
