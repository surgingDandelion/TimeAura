# TimeAura 下一阶段开发 Backlog

## 1. 文档目的

本文件用于承接当前桌面端第一轮可体验版本之后的下一阶段开发工作，重点解决：

- 已有页面看起来可用，但主线功能仍未完全闭环的问题
- 删除 / 回收站、快速新增解析、AI 报告动作、AI 回退策略等高优先级缺口
- 让后续开发、测试、产品验收可以直接按优先级推进

说明：

- `P0`：必须优先完成，直接影响核心体验
- `P1`：V1 强建议完成，影响产品完整度
- `P2`：第二阶段增强项，偏产品化与设置中心

---

## 2. 当前阶段结论

当前桌面端已具备以下可体验基础：

- 主工作台三栏结构
- 标签管理
- 提醒条与批量改期
- AI 通道配置
- AI 报告生成
- 浅色 / 暗色主题切换

但以下能力尚未完全闭环：

- 删除仅停留在软删除动作，缺少回收站页面、恢复与彻底删除链路
- 快速新增尚未实现 `#标签 / !P1 / @时间` 智能解析
- 模糊检索尚未真正覆盖标签名
- 计划时间主线未完整进入详情与新增交互
- AI 报告页头部动作与导出能力未打通
- AI 执行失败后的回退策略尚未符合 PRD

---

## 3. P0：必须优先完成

### P0-1 删除闭环

目标：

- 让“删除”从一个接口动作变成可恢复、可追溯、可清理的完整桌面端闭环

任务拆解：

- [ ] 统一记录删除语义为软删除
- [ ] 为 `RecordService` 增加恢复记录能力
- [ ] 为 `RecordService` 增加彻底删除能力
- [ ] 为 `RecordService` 增加清空回收站能力
- [ ] 为 Mock / SQLite repository 增加恢复、硬删除、清空能力
- [ ] 左下角回收站入口启用
- [ ] 新增回收站页面，展示已删除记录
- [ ] 支持单条恢复
- [ ] 支持单条彻底删除
- [ ] 支持清空回收站
- [ ] 删除 / 恢复 / 清空后联动刷新主列表、提醒条、侧栏计数
- [ ] 当前正在查看的记录若被删除，详情区自动收起
- [ ] 补删除与回收站相关测试

验收标准：

- 用户删除记录后，不会直接丢失数据
- 用户可在回收站恢复记录到工作台
- 用户可在回收站彻底删除记录
- 回收站清空后页面与计数即时刷新

### P0-2 快速新增智能解析

目标：

- 让快速新增真正体现“低输入成本”的 TimeAura 价值

任务拆解：

- [ ] 解析 `#标签`
- [ ] 解析 `!P1 / !P2 / !P3 / !P4`
- [ ] 解析常用时间短语，如 `@今天 18:00 / @明早 09:00 / @明天下午 3 点`
- [ ] 将解析结果写入标签、优先级、截止时间或计划时间
- [ ] 未解析出的文本自动回落为标题
- [ ] 继续保持回车连续新增体验
- [ ] 补解析规则测试

验收标准：

- 用户输入一行自然文本即可生成结构化记录
- 创建结果与输入意图基本一致

### P0-3 模糊检索补全

目标：

- 让“模糊检索标题、正文、标签”名副其实

任务拆解：

- [ ] SQLite 查询增加标签名匹配
- [ ] Mock 查询增加标签名匹配
- [ ] 保持与现有标题 / 正文搜索语义一致
- [ ] 补标签搜索测试

验收标准：

- 仅输入标签名也能搜到对应记录
- Mock 与 SQLite 结果一致

### P0-4 计划时间主线补齐

- [ ] 排序与提醒逻辑补计划时间联动
- [ ] 补相关测试

验收标准：

- 用户可直接维护计划时间
- 今天 / 计划视图结果更符合直觉
