# TimeAura 开发任务拆解 Backlog

## 1. 文档目的

本文件用于把 TimeAura 当前阶段的开发任务拆成可执行 backlog，方便：

- 产品排期
- 前端开发拆任务
- AI 编码模型按批次实现
- 评估 V1 优先级

说明：

- 以下按优先级分为 `P0 / P1 / P2`
- `P0` 为必须先完成的核心闭环
- `P1` 为 V1 强建议完成
- `P2` 为增强项或 polish 项

## 6. 建议任务粒度

推荐每个开发任务满足以下条件：

- 一个任务只改一类问题
- 一个任务可在半天到一天内完成
- 一个任务有明确验收标准
- 一个任务最好对应一个或少量文件域

不建议把以下内容合成一个大任务：

- “做完主工作台”
- “做完所有 AI 功能”
- “统一所有状态”

建议拆成：

- “实现 TaskRow active / selected / done 状态”
- “实现 ReminderBanner 展开态”
- “实现 ChannelFormPanel 基础字段”

## 8. 简版结论

如果你要直接进入开发，最实用的做法是：

1. 先按 `P0` 跑通主工作台闭环
2. 再按 `P1` 补齐提醒条、标签管理、AI 报告、AI 通道
3. 最后按 `P2` 做主题、状态、控件与细节 polish

这样可以保证 TimeAura 先有“可用的产品主干”，再逐步补齐“像专业桌面工具”的完成度。
